# 电位网格三维可视化设计草案

## 1. 目标概述

本设计旨在基于 `CircuitJS1` 现有仿真内核，新增一个独立的三维电位可视化窗口，将电路中各个节点的电位高低，以黑色背景上的连续三维网格曲面形象展示出来。

设计目标不是替代原有二维电路图，而是在其基础上增加一种新的观察维度，让用户能够直观看到：

- 哪些区域是高电位
- 哪些区域是低电位
- 各节点电位如何随着仿真动态变化
- 某个节点变化后，整个电路电位分布如何整体联动

该功能第一版定位为：

- 独立新窗口
- 仅基于导线连通后的真实电气节点电位
- 地线作为 `0V` 基准面
- 全电路区域铺满连续曲面
- 节点亮点保留
- 每秒约 `20-30` 次刷新

## 2. 目标画面

目标画面包含以下核心视觉元素：

- 黑色背景
- 一张覆盖整个电路区域的三维网格曲面
- 网格曲面的高度对应节点电位
- 高电位区域抬升，形成山峰或山脊
- 低电位区域下沉，形成山谷或洼地
- 未被支路经过的区域仍保留平坦网格
- 支路经过器件时，两端电位通过连续曲面过渡，不出现断裂
- 真实节点位置以亮点标出
- 保留坐标轴和颜色图例
- 支持鼠标旋转、缩放、平移

## 3. 用户已经确认的需求

### 3.1 显示范围

- 显示整个电路区域，不只沿导线绘制狭窄带状曲面
- 没有支路经过的区域显示为水平不变的网格

### 3.2 进入方式

- 第一版从原程序中打开一个单独的三维窗口
- 不直接破坏原二维编辑界面

### 3.3 电位基准

- 以地线为 `0V`
- 高于地线的电位显示为高于水平面的地形
- 低于地线的电位显示为低于水平面的地形

### 3.4 曲面连续性

- 整个网格必须连续
- 支路经过器件时，器件两端采用连续曲面过渡
- 不能把器件位置处理成断点

### 3.5 刷新策略

- 不追求每个仿真步都重绘
- 采用限帧策略
- 第一版目标刷新频率约 `20-30 FPS`

### 3.6 节点表现

- 在曲面上保留节点亮点
- 节点亮点用于告诉用户“真实电气节点”的离散位置

### 3.7 高度缩放

- 第一版采用自动缩放
- 根据当前电路中的最大/最小电位范围自动调整曲面高度

### 3.8 辅助信息

- 保留三维坐标轴
- 保留颜色图例

## 4. 技术可行性判断

该功能具备实现基础，因为 `CircuitJS1` 源码里本来就有“节点”和“节点连接”的概念，而不是单纯画图。

现有关键基础包括：

- [CircuitNode.java](file:///D:/文件/电气电子/电路仿真demo/circuitjs1-master/src/com/lushprojects/circuitjs1/client/CircuitNode.java)
- [CircuitNodeLink.java](file:///D:/文件/电气电子/电路仿真demo/circuitjs1-master/src/com/lushprojects/circuitjs1/client/CircuitNodeLink.java)
- [CircuitElm.java](file:///D:/文件/电气电子/电路仿真demo/circuitjs1-master/src/com/lushprojects/circuitjs1/client/CircuitElm.java)
- [CirSim.java](file:///D:/文件/电气电子/电路仿真demo/circuitjs1-master/src/com/lushprojects/circuitjs1/client/CirSim.java)

这意味着：

- 节点不是靠识别屏幕像素推断
- 电位值不是靠视觉估计
- 节点、电位、器件连接，都来自仿真内核内部数据

## 5. 关键数据来源

第一版的核心数据分三类：

### 5.1 节点拓扑

来自：

- `CircuitNode`
- `CircuitNodeLink`
- 各元件的 `nodes[]`

用途：

- 确定有哪些电气节点
- 确定每个节点连到哪些元件
- 确定节点之间的拓扑关系

### 5.2 二维几何位置

来自：

- 各器件端点位置 `getPost(n)`
- 导线和元件的屏幕坐标

用途：

- 为三维网格提供 `x/y` 平面位置
- 把电气节点映射回二维电路图上的真实位置

### 5.3 节点电位值

来自：

- 各元件的 `volts[]`
- 仿真求解结束后的节点电压结果

用途：

- 生成三维高度 `z`
- 计算颜色映射
- 驱动曲面的动态变化

## 6. 视觉模型定义

第一版采用“势场地形图”模型。

### 6.1 坐标系统

- `x`：二维电路图横向位置
- `y`：二维电路图纵向位置
- `z`：节点电位映射后的高度

### 6.2 高度定义

- 地线节点高度为 `0`
- 其余节点高度按电位值经过缩放函数得到

第一版推荐公式：

```text
z = scale * normalize(Vnode)
```

其中：

- `Vnode` 为节点电位
- `normalize()` 根据当前电路最大正电位和最小负电位做自动缩放
- `scale` 为场景整体高度因子

### 6.3 颜色定义

建议第一版采用三段式配色：

- 低电位：深蓝到亮蓝
- 接近零电位：深灰或中性暗色
- 高电位：橙红到亮红

推荐规则：

```text
V > 0  -> 暖色梯度
V = 0  -> 中性暗色
V < 0  -> 冷色梯度
```

### 6.4 节点亮点

- 每个真实节点位置放置一个发光点
- 颜色可跟随节点电位颜色
- 尺寸略大于普通网格交点

## 7. 连续曲面的生成策略

这是本功能最关键的视觉层设计。

### 7.1 为什么需要插值

仿真内核给出的真实物理量是“离散节点电位”，不是整片连续表面。

因此连续网面的本质是：

- 节点高度来自物理结果
- 节点之间的连续面由可视化层进行空间插值

### 7.2 第一版建议方案

采用“二维规则网格 + 节点势场插值”的方式：

1. 以整个电路外接矩形为范围
2. 在该范围内生成规则二维网格
3. 对网格上每个采样点，根据附近节点电位进行插值
4. 生成一个连续高度场
5. 将高度场转成三角面片网格

### 7.3 对支路区域的强调

为了让支路更像“山脊/山谷”，可以在第一版中引入一套权重机制：

- 靠近导线和器件中心线的区域，优先受该支路两端节点影响
- 远离支路的区域，高度逐渐回归平滑背景

结果上会形成：

- 支路附近地形起伏更明显
- 空白区域网格更平缓

### 7.4 器件处连续过渡

用户已明确要求支路经过器件时不断裂。

因此第一版规则为：

- 器件不作为“绝缘切断面”处理
- 器件图形所在区域，采用其两端节点电位做连续过渡
- 对于存在电压降的器件，这一段会表现为坡面

## 8. 动态更新策略

### 8.1 更新频率

第一版目标：

- 仿真内部仍按原有频率计算
- 三维视图以 `20-30 FPS` 重绘

### 8.2 数据同步方式

建议采用“拉取式快照”：

- 仿真主循环持续更新节点电位
- 三维视图每次重绘时读取当前电位快照
- 避免每个仿真步都强制触发三维窗口重建

### 8.3 稳定性处理

为防止画面抖动，可加入轻微时间平滑：

```text
Zdisplay = lerp(Zprevious, Zcurrent, alpha)
```

但第一版应保持平滑量较小，避免掩盖真实变化。

## 9. 窗口架构建议

### 9.1 第一版结构

- 主窗口：保持原有二维电路编辑与仿真
- 新窗口：三维电位网格视图

### 9.2 同步关系

主窗口负责：

- 电路编辑
- 电路仿真
- 节点电位计算

新窗口负责：

- 三维渲染
- 交互旋转/缩放/平移
- 电位快照显示

### 9.3 第一版不建议做的事

- 不建议一开始把三维视图嵌入主界面右侧
- 不建议第一版就做复杂多面板联动
- 不建议第一版就支持所有导出功能

## 10. 最小可用版本定义

第一版最小可用目标建议定为：

### 必做

- 能打开一个独立三维窗口
- 能读取当前电路节点电位
- 能显示整个电路范围的连续网格曲面
- 能显示高低电位颜色变化
- 能显示节点亮点
- 能以地线为零平面
- 能鼠标旋转查看
- 能动态刷新

### 暂不必做

- 电压数值文字悬浮标签
- 视频录制
- 导出三维模型
- 高级自定义材质
- 多种插值算法切换
- 多窗口同步编辑

## 11. 第一版开发风险

### 11.1 物理风险

低。

原因：

- 电位计算仍由原内核负责
- 新功能主要是读取和显示，不重写求解器

### 11.2 视觉风险

中等。

原因：

- 连续曲面如果插值不当，可能会看起来“好看但不直观”
- 高度自动缩放若不稳，可能导致画面忽高忽低

### 11.3 性能风险

中等。

原因：

- 三维网格每秒重建 `20-30` 次时，需要控制网格分辨率
- 电路很复杂时，插值和渲染负载会升高

## 12. 第一版开发顺序建议

建议严格按下面顺序推进：

1. 验证如何从当前仿真中提取节点位置和节点电位
2. 建立一个静态三维测试窗口
3. 先用简单测试电路生成静态高度场
4. 再加入颜色映射
5. 再加入节点亮点
6. 再加入实时刷新
7. 最后再调视觉细节和交互体验

## 13. 第一版验收标准

如果出现下面现象，就说明第一版方向成功：

- 改变电源值时，整张三维网面随之变化
- 某支路导通或断开时，相关区域地形明显变化
- 高电位区域能直观看出更高、更红
- 低电位区域能直观看出更低、更蓝
- 地线附近区域明显接近零平面
- 用户可以通过旋转观察整个电位分布

## 14. 当前结论

这个功能是可以做的。

它不是重写电路仿真器，而是在现有正确仿真结果的基础上，构建一个新的三维电位可视化层。

第一版的核心关键词可以总结为：

- 正确节点
- 连续曲面
- 地线零面
- 山峰山谷
- 动态联动
- 独立窗口
- 立体可旋转

这份草案作为后续编码前的设计基线使用。
